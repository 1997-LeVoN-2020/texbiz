## Шаги

1. **Зарегистрировать страницу в `PAGES`** (`app.py`) — добавить новый словарь в список с полями:
   - `id` — уникальный идентификатор для `url_for()`
   - `url_path` — путь относительно корня (например `nastroika-kass/`), с завершающим `/`
   - `template` — путь к шаблону относительно `src/templates/` (например `pages/nastroika-kass.html`)
   - `title`, `description` — уникальные, под целевые ключевые фразы
   - `og`, `og_type`, `og_title`, `og_description` — если нужны OG-теги
   - `priority`, `changefreq` — для sitemap.xml
2. **Создать шаблон** в `src/templates/pages/` (или `src/templates/pages/blog/` для статьи), расширяющий `base.html`, по образцу существующих страниц того же типа (услуга — смотреть на `1c-avtomatizaciya-otelya.html`, статья — на существующие файлы в `blog/`).
3. **Добавить пункт меню**, если это новая страница услуги — в `_header.html` (dropdown «Услуги»).
4. **Добавить JSON-LD разметку** в шаблон — `Service` для услуги, `Article` для статьи блога (см. skill `link-and-json-ld-check` за требованиями к полям).
5. **Проверить**: перезапустить/дождаться автоперезагрузки Flask-приложения (`python app.py`, debug-режим сам подхватывает изменения в `app.py`), открыть новый URL в браузере и проверить `/sitemap.xml` — страница должна там появиться автоматически (генерируется из `PAGES` на лету).
6. **Проверить ссылки**: использовать `url_for('новый_id')` в остальных шаблонах, где нужна ссылка на эту страницу — не хардкодить путь руками.

## Не забыть

Контент страницы — если нужен качественный копирайтинг под тон сайта, привлеки агента `content-writer`. SEO-поля в `PAGES` (title/description) можно затем прогнать через агента `seo-auditor` на предмет уникальности и качества.
