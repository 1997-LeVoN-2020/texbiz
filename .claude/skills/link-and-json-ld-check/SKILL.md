## Битые внутренние ссылки

1. Убедиться, что `dist/` собран и актуален (`python build.py`, см. скилл `build-and-preview`).
2. Пройти по всем HTML-файлам в `dist/`, собрать все `<a href="...">` с относительными/локальными путями (не `http(s)://`, не `mailto:`, не `tel:`).
3. Для каждой такой ссылки проверить, что целевой файл (`index.html` в соответствующей директории, с учётом чистых URL без `.html`) существует в `dist/`.
4. Отдельно проверить: `sitemap.xml` содержит только реально существующие в `dist/` страницы, `robots.txt` ссылается на актуальный `sitemap.xml`.

## Валидность JSON-LD

1. Извлечь содержимое всех `<script type="application/ld+json">` из страниц `dist/`.
2. Проверить, что каждый блок — валидный JSON (например через `json.loads` в Python).
3. Сверить тип разметки со страницей, на которой он находится:
   - главная — `ProfessionalService` + `FAQPage`
   - `booking/` — `SoftwareApplication`
   - страницы услуг (`1c-avtomatizaciya-otelya/`, `onlain-kassy-dlya-gostinicy/`, `nastroika-serverov-otelya/`, `integraciya-zamkovyh-sistem/`) — `Service`
   - статьи блога — `Article`
4. Проверить обязательные поля каждого типа (например `Article` требует `headline`, `datePublished`, `author`).

## Формат результата

Список найденных проблем с указанием файла и конкретной ссылки/блока. Если проблем нет — явно сообщить об этом с указанием, сколько страниц и ссылок проверено (для сверки с зафиксированным в README статусом «0 битых ссылок, весь JSON-LD валиден»).
