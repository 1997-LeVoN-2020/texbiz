## Битые внутренние ссылки

1. Запустить приложение локально (`python app.py`, см. `.claude/launch.json` конфиг `flask-app`, порт 5000).
2. Для каждой страницы из `PAGES` (`app.py`) запросить её URL и собрать все `<a href="...">` с относительными/локальными путями (не `http(s)://`, не `mailto:`, не `tel:`).
3. Для каждой такой ссылки сделать запрос и убедиться, что ответ не 404 (все внутренние ссылки строятся через `url_for()`, поэтому обычно ссылаются на реально зарегистрированный в `PAGES` маршрут — но стоит проверить фактически, а не полагаться только на код).
4. Отдельно проверить `GET /sitemap.xml` — там должны быть только реально существующие маршруты, и `GET /robots.txt` — должен ссылаться на `/sitemap.xml`.

## Валидность JSON-LD

1. Для каждой страницы получить HTML и извлечь содержимое всех `<script type="application/ld+json">`.
2. Проверить, что каждый блок — валидный JSON (например через `json.loads` в Python).
3. Сверить тип разметки со страницей, на которой он находится:
   - главная — `ProfessionalService` + `FAQPage`
   - `booking/` — `SoftwareApplication`
   - страницы услуг (`1c-avtomatizaciya-otelya/`, `onlain-kassy-dlya-gostinicy/`, `nastroika-serverov-otelya/`, `integraciya-zamkovyh-sistem/`) — `Service`
   - статьи блога — `Article`
4. Проверить обязательные поля каждого типа (например `Article` требует `headline`, `datePublished`, `author`).

## Формат результата

Список найденных проблем с указанием страницы и конкретной ссылки/блока. Если проблем нет — явно сообщить об этом с указанием, сколько страниц и ссылок проверено.
