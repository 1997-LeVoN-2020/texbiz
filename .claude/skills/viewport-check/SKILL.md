## Известный прецедент

На этом проекте уже был найден и исправлен баг горизонтального переполнения на 320-375px:
- `.hero::before` (декоративное псевдо-пятно) вылезал на 160px за пределы секции → исправлено `.hero { overflow: hidden }`
- грид-колонки с голым `1fr` распирались по min-content длинных слов → исправлено на `minmax(0, 1fr)`

При каждой проверке в первую очередь смотреть именно на эти паттерны в новых/изменённых CSS-правилах.

## Шаги

1. Запустить приложение локально (`python app.py`, см. `.claude/launch.json` конфиг `flask-app`, порт 5000).
2. Через headless-браузер открыть каждую из страниц на viewport 320px, 375px, 768px, 1280px.
3. На каждом viewport сравнить `document.documentElement.scrollWidth` и `document.documentElement.clientWidth` — если `scrollWidth > clientWidth`, есть горизонтальное переполнение.
4. При обнаружении переполнения — найти виновный элемент (обычно через бинарный поиск по DOM: скрывать/сужать подозрительные блоки и смотреть, пропадает ли переполнение) и предполагаемое CSS-правило.
5. Сделать скриншот проблемного viewport для визуальной проверки.

## Формат результата

Для каждой проверенной страницы — список viewport, на которых найдено переполнение (если есть), с указанием предполагаемого селектора. Если всё чисто — подтверждение с перечислением проверенных страниц и viewport.
