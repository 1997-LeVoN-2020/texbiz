## Шаги

1. Установить зависимости (один раз или после изменения `requirements.txt`):
   ```
   pip install -r requirements.txt
   ```
2. Собрать сайт:
   ```
   python build.py
   ```
   Результат — `dist/` с чистыми URL, `sitemap.xml`, `robots.txt`, `.htaccess`, `404.html`.
3. Поднять локальный HTTP-сервер на `dist/` (например `python -m http.server 8000` из директории `dist/`).
4. Открыть в браузере `http://localhost:8000/` и убедиться, что главная страница и хотя бы одна страница услуги открываются без ошибок.

## Если сборка падает

Единственный источник правды по страницам — реестр `PAGES` в `build.py`. Проверь, что:
- у страницы, на которой падает сборка, есть корректная запись в `PAGES` (`id`, `url_path`, `template`)
- шаблон, указанный в `template`, существует в `src/templates/pages/`
- используемые в шаблоне `url_for('some_id')` ссылаются на существующие `id` из `PAGES`
