## Команда

```
python package.py
```

Собирает сайт (`build.py`) и упаковывает `builds/texbiz-site-<VERSION>.zip` + `builds/texbiz-pyapp-<VERSION>.zip`. `<VERSION>` берётся из файла `VERSION` в корне репозитория.

`package.py` **откажется** собирать архив, если файлы с текущей версией уже есть в `builds/` — старые версии никогда не перезаписываются и не удаляются. Перед новой сборкой нужно поднять версию в `VERSION` (семвер, например `1.0.0` → `1.0.1`/`1.1.0`).

`builds/` в `.gitignore`, архивы не коммитятся — но локально накапливается история версий для отката.

## Куда что распаковывать (важно для этого хостинга)

- **`texbiz-site-<VERSION>.zip`** (содержимое `dist/`, без обёрточной папки) → напрямую в директорию домена в ISPmanager (`/var/www/<пользователь>/data/www/tex-biz.ru`).
- **`texbiz-pyapp-<VERSION>.zip`** (содержимое `src/pyapp/`) → в подпапку `mailapp/` **внутри той же** директории домена — на этом тарифе отдельной директории для Python-приложений нет, поэтому статика и Python-приложение физически лежат в одном корне (см. README, раздел «2. Python-приложение для формы»).

## После загрузки

Свериться, что `MAIL_ENDPOINT` в `src/static/script.js` (по умолчанию `/mailapp/send`) совпадает с реальным путём Python-приложения, настроенным в ISPmanager. Если путь другой — поменять константу в `script.js`, поднять `VERSION` и пересобрать (`python package.py`).
