Два уровня поиска: сначала локально (быстро, бесплатно, ничего не ставить), и только если там пусто — наружу на GitHub.

## Фаза 1 — локально (всегда сначала)

1. **Прочитать фронтматтер всех агентов и скиллов этого проекта заново** (не полагаться на память — список растёт):
   - `.claude/agents/*.md` — поле `description` в YAML-фронтматтере каждого файла.
   - `.claude/skills/*/SKILL.md` — то же самое.
2. **Сопоставить формулировку задачи** с `description` каждого агента/скилла — они специально написаны с конкретными триггер-фразами ("использовать когда просят..."), сверяться в первую очередь по ним.
3. Если подходящий агент — вызвать через инструмент Agent (`subagent_type` соответствует имени файла в `.claude/agents/`). Если подходящий скилл — вызвать через Skill.
4. Если ничего локального не подошло — переходить к фазе 2, не молчать и не выдумывать решение самому.

## Фаза 2 — поиск на GitHub (если фаза 1 ничего не дала)

1. Сформулировать поисковый запрос по сути задачи + `"Claude Code skill"` или `SKILL.md` (например: `WebSearch("Claude Code skill SKILL.md <тема задачи> github")`).
2. Полезные отправные точки: официальный репозиторий `anthropics/skills` на GitHub, а также маркетплейсы/коллекции сторонних skills для Claude Code — искать через обычный веб-поиск, не угадывать конкретные URL.
3. Открыть/прочитать найденный кандидат (`WebFetch` по ссылке на репозиторий/`SKILL.md`), проверить, что это действительно формат Claude Code skill (YAML-фронтматтер `name`/`description` + инструкции), а не что-то по касательной.
4. **Ничего не устанавливать и не копировать в проект без явного подтверждения пользователя** — скачивание/добавление стороннего кода в репозиторий требует явного разрешения (см. правила безопасности). Показать пользователю, что нашёл (название, репозиторий, что делает), и спросить, добавлять ли.
5. Если и на GitHub ничего подходящего не нашлось — тогда честно предложить оформить новый агент/скилл под этот проект (см. эвристику ниже), а не тянуть задачу молча мимо системы.

## Как решить: новый агент, новый скилл, или ничего не заводить

- **Скилл** — если это процедура без суждения (чёткая последовательность шагов, повторяемая один в один).
- **Агент** — если нужен анализ/суждение, отдельная точка зрения, или изоляция контекста/ограничение набора инструментов (например read-only ревью, которое не должно случайно ничего изменить).
- **Ничего не заводить** — если это разовая узкая задача, не похожая на то, что повторится.

## Формат ответа

Назвать конкретный агент/скилл (или несколько, если задача составная) и **сразу его вызвать**, а не просто перечислить варианты и ждать подтверждения — если только выбор неочевиден между двумя близкими кандидатами, или дело дошло до фазы 2 (там подтверждение обязательно перед добавлением чего-либо в проект).
