# ТЕХБИЗ — сайт

Многостраничный сайт компании ТЕХБИЗ («технологии для бизнеса») — комплексная автоматизация гостиничного бизнеса под ключ. Собирается Python-скриптом (`build.py` + Jinja2) из шаблонов в `src/` в статический `dist/`, который и выкладывается на хостинг. Никакого PHP в проекте нет — ни в сборке, ни на сервере.

## Назначение

Привлечение клиентов и конверсия входящего трафика в заявки на услуги автоматизации гостиниц, отелей, мини-отелей и апарт-отелей, а также отдельная продажа модуля онлайн-бронирования как самостоятельного продукта.

## Технологии

- **Сборка**: Python 3 + Jinja2 (`build.py`). Один реестр страниц (`PAGES` в `build.py`) — единственный источник правды по каждому URL, title, description и SEO-настройкам. Шапка/футер вынесены в шаблоны-партиалы, не дублируются по файлам.
- **Вывод**: чистый статический HTML/CSS/JS (`dist/`) — без рантайм-зависимостей, подходит для любого хостинга.
- **Форма заявки**: Flask-приложение (`src/pyapp/`), запускается на хостинге как отдельный Python-процесс (WSGI/Passenger), не PHP.
- **Стили/скрипты**: CSS3 без фреймворков, JavaScript vanilla — лежат в `src/static/`, копируются в `dist/` как есть.

## Как собрать

```
pip install -r requirements.txt
python build.py
```

Результат — папка `dist/` с чистыми URL (`/booking/`, `/1c-avtomatizaciya-otelya/`, ...), `sitemap.xml`, `robots.txt` и `.htaccess`. Содержимое `dist/` целиком заливается в корень хостинга.

## Чистые адреса без .html

Каждая страница — отдельная папка с `index.html` (`booking/index.html`, `blog/1c-avtomatizaciya-otelya-s-chego-nachat/index.html` и т.д.), поэтому адрес получается без расширения — так работает на любом статическом хостинге без специальных настроек сервера. `.htaccess` дополнительно 301-редиректит старые адреса вида `/booking.html` на `/booking/`, если они где-то проиндексированы или сохранены в закладках.

Внутренние ссылки не хардкодятся руками — `url_for('booking')`, `url_for('svc_1c')` и т.д. в шаблонах, реализация в `build.py`, сама считает нужное количество `../` по глубине текущей страницы.

## Форма заявки — отдельное Python-приложение

`src/pyapp/` — Flask-приложение с одним маршрутом `POST /send`: проверяет honeypot-поле, обязательные поля и отправляет письмо на `info@tex-biz.ru` через `smtplib`. Разворачивается на хостинге отдельно от статического сайта (ISPmanager запускает Python-приложения через Passenger, `passenger_wsgi.py` — стандартная точка входа для этой схемы).

Пока приложение не развёрнуто, форма на сайте показывает понятное сообщение об ошибке вместо ложного «заявка отправлена» — есть автоматический fallback на клиенте.

## Деплой на хостинг (reg.ru, ISPmanager)

### 0. Собрать архивы для загрузки

```
python build.py
python -c "
import zipfile, os
def zip_dir(src, out):
    with zipfile.ZipFile(out, 'w', zipfile.ZIP_DEFLATED) as zf:
        for root, _, files in os.walk(src):
            for f in files:
                full = os.path.join(root, f)
                zf.write(full, os.path.relpath(full, src))
zip_dir('dist', 'texbiz-site.zip')
zip_dir('src/pyapp', 'texbiz-pyapp.zip')
"
```

Получаются `texbiz-site.zip` (содержимое `dist/`, без обёрточной папки — распаковывать прямо в директорию домена) и `texbiz-pyapp.zip` (содержимое `src/pyapp/` — распаковывать в подпапку `mailapp/` внутри той же директории домена, см. раздел «2. Python-приложение для формы» — отдельной директории для Python-приложений на этом тарифе нет). Оба в `.gitignore`, не коммитятся.

### 1. Статический сайт

1. Локально: `pip install -r requirements.txt && python build.py` → получаем `dist/` (или готовый `texbiz-site.zip`, см. шаг 0).
2. В ISPmanager: «Домены» / «WWW-домены» → выбрать `tex-biz.ru` → посмотреть путь к директории сайта (обычно `/var/www/<пользователь>/data/www/tex-biz.ru`).
3. Залить содержимое `dist/` (не саму папку, а файлы из неё) в эту директорию — через SFTP (например, FileZilla с данными доступа из ISPmanager), встроенный файловый менеджер ISPmanager, или распаковать там `texbiz-site.zip`.
4. Проверить, что `.htaccess` реально загрузился (файловые менеджеры иногда скрывают dotfiles) и что домен уже привязан к SSL-сертификату в ISPmanager (иначе редирект на HTTPS из `.htaccess` уведёт на несуществующий сертификат).

> **Грабли (было на этом проекте):** HTTPS-редирект внутри `.htaccess` (`RewriteCond %{HTTPS} off`) вызвал `ERR_TOO_MANY_REDIRECTS` на reg.ru — за проксирующим/WAF-слоем этого хостинга `%{HTTPS}` не отражает реальный протокол запроса, и правило зацикливалось само на себя. В ISPmanager у сайта уже есть своя галочка «перенаправлять http-запросы в HTTPS» — HTTPS-редирект в `.htaccess` намеренно убран, чтобы не дублировать и не ломать эту логику; в `.htaccess` остался только редирект `www` → без `www` (он безопасен, не зависит от `%{HTTPS}`).

### 2. Python-приложение для формы (`src/pyapp/`)

> На этом тарифе хостинг даёт только один корень для домена — отдельной директории для Python-приложений нет. Поэтому `src/pyapp/` разворачивается не рядом, а **внутри** той же директории сайта, подпапкой `mailapp/` (`/var/www/<пользователь>/data/www/tex-biz.ru/mailapp`) — то есть внутри директории из шага 1.2.

1. В ISPmanager открыть раздел **«Python»** (в списке модулей слева или на странице домена) → **«Создать»**.
2. Указать: домен `tex-biz.ru`, путь приложения `/mailapp` (это и станет `MAIL_ENDPOINT`), версию Python 3. В качестве директории для файлов приложения указать подпапку `mailapp` внутри директории сайта из шага 1.2 (а не отдельную/внешнюю директорию, которую ISPmanager может предложить по умолчанию).
3. ISPmanager создаст пустое приложение с virtualenv и своим `passenger_wsgi.py`-заглушкой в этой подпапке — её нужно заменить содержимым `src/pyapp/` (загрузить `app.py`, `passenger_wsgi.py`, `requirements.txt` через SFTP/файловый менеджер, затерев заглушку — либо распаковать туда `texbiz-pyapp.zip`, см. шаг 0). Содержимое `dist/` (шаг 1.3) и содержимое `texbiz-pyapp.zip` в итоге лежат в одной директории: `mailapp/` — подпапка внутри неё же.
4. Установить зависимости в virtualenv приложения. В ISPmanager обычно есть кнопка «Установить зависимости» / «pip install -r requirements.txt» на странице приложения; если её нет — через SSH: `<путь_к_venv_приложения>/bin/pip install -r requirements.txt`.
5. Если хостинг требует авторизованный SMTP, а не локальный релей — задать переменные окружения `SMTP_HOST`, `SMTP_PORT`, `SMTP_USER`, `SMTP_PASSWORD` в настройках Python-приложения в ISPmanager.
6. Перезапустить приложение (кнопка «Перезапустить» / «Restart» на странице приложения в ISPmanager).
7. Свериться, что `MAIL_ENDPOINT` в `src/static/script.js` совпадает с реальным путём приложения из шага 2 (по умолчанию в коде — `/mailapp/send`; если в ISPmanager указали другой путь, поменять константу и пересобрать: `python build.py`, перезалить `dist/styles.css`... то есть `dist/script.js`).
8. Проверить вручную: открыть сайт, отправить тестовую заявку через форму, убедиться, что письмо дошло на `info@tex-biz.ru`.

Точные названия кнопок могут отличаться в зависимости от версии ISPmanager — общая механика (Passenger + virtualenv + `passenger_wsgi.py`) одна и та же.

## Функциональные требования

- Адаптивность от 320px.
- Фиксированная шапка с плавным скроллом к блокам (на главной) и мобильное меню (бургер) на всех страницах.
- Анимация появления блоков при прокрутке.
- Валидация формы на клиенте (обязательные поля, формат телефона) и отправка на Flask-бэкенд, с понятным сообщением об ошибке, если он недоступен.
- Калькулятор ориентировочного бюджета (расчёт на клиенте, только на главной).

## SEO и индексация

- Уникальные `title`, `meta description`, `canonical`, Open Graph и Twitter Card — на каждой странице отдельно, задаются один раз в реестре `PAGES` (`build.py`).
- JSON-LD микроразметка: `ProfessionalService` + `FAQPage` на главной, `SoftwareApplication` на странице модуля бронирования, `Service` на страницах услуг, `Article` на статьях блога.
- `sitemap.xml` и `robots.txt` генерируются билдом из того же реестра `PAGES` — не могут разойтись с реальными страницами.
- Ключевые фразы: автоматизация отеля 1С, настройка касс для гостиницы, интеграция замковой системы, настройка сервера для отеля, модуль онлайн-бронирования для отеля.
